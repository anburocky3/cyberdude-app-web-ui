# Contributing

Thanks for your interest in contributing to **cyberdude-app-web-ui**!  
This guide keeps things simple so you can get started quickly.

---

## What to read first

- Read the main `README.md` to understand the project purpose, setup, and folder structure.
- Check existing issues and pull requests to avoid duplicate work.
- Keep changes small, focused, and well-documented.

---

## Reporting issues

If you find a bug or want to suggest an improvement:

- Open an issue in the GitHub repository.
- Clearly describe:
  - What happened
  - What you expected to happen
  - Steps to reproduce
  - Screenshots or screen recordings (if applicable)
- Mention browser, device, and OS if it's a UI-related issue.

---

## Styling guidelines

- Follow the existing styling approach (Tailwind CSS / CSS Modules / Styled Components).

- Keep class names clean and consistent.

- Ensure responsiveness across devices.

- Avoid inline styles unless necessary.
